# 百灵 Bailin · 阶段性路线图（ROADMAP v0.1）

> 配套：[PRD.md](PRD.md) §11 / [TECH-ROUTE.md](TECH-ROUTE.md) / [CHARACTER-PROTOCOL.md](CHARACTER-PROTOCOL.md)
> 目的：把"主动建议 / 养成 / 角色市场 / 深度造人 / 多模态 / 跨端"放进可执行的阶段中，控制每阶段的不可逆决策

---

## 0. 路线图原则

1. **每阶段只押一个主题**：宁可一阶段只做一件事，也不要 5 件半成品并行
2. **每阶段都有可演示的用户价值**：内部重构不构成独立版本
3. **不可逆决策前置**：影响协议 / 数据 / 沙箱 / 安全的决策必须在 MVP 与 v1.0 之间想清楚
4. **路径可回滚**：如果某阶段验证失败，下一阶段必须能"绕过"或"撤回"
5. **不锁死时间，先锁死顺序**：版本号决定先后，不承诺月份

---

## 2. v0.x · MVP（核心闭环）

### 2.1 主题

> "证明：用户能创建一个角色，让它出现在桌面，并完成第一次有质感的对话。"

### 2.2 范围（详见 [PRD §4.2](PRD.md)）

- 首启 + Key 配置
- 60 秒造人（快速版女娲流程）
- 程序化像素桌宠（DSL 模式）
- 桌面常驻 + 唤起聊天
- 轻量用户画像记忆
- 6 个内置示例角色（3 实用 + 3 情感）
- 基础安全 / 免责
- Windows 单平台

### 2.3 不做

- 深度造人 / 联网调研
- 关系记忆 / 好感度
- 主动提醒
- 角色市场 / 导入导出
- JS 沙箱默认开启
- macOS / Linux
- 多角色同时在桌面

### 2.4 成功指标

- **创建完成率 ≥ 60%**（从启动到第一只角色上桌）
- 首次对话发起率 ≥ 80%
- 次日召唤率 ≥ 30%

### 2.5 退出门（进入 v1.0 前必须 done）

- [ ] CharacterCard / SpriteProgram Schema 稳定（未来仅加字段，不破坏字段）
- [ ] SpriteCodeGen 安全沙箱通过红队测试（10+ 恶意代码尝试全部被拒）
- [ ] 6 个示例角色的人格 + 形象都过"内部品味关"
- [ ] 至少 30 名外部用户跑通完整首启 + 造人

## 3. v1.0 · 体验提升

### 3.1 主题

> "让一只桌宠从'能用'升级到'我愿意天天召唤'。"

### 3.2 关键能力

#### 3.2.1 深度造人模式

- 在快速版之外，提供"深度模式"
- 时长 10~20 分钟，单角色单次更高质量
- 如用户的 LLM 支持联网工具调用，自动启用网络补充信息
- 输出比快速版多：心智模型补充 + 内在张力 + 智识谱系（入卡）
- 失败可降级到快速版结果

#### 3.2.2 形象丰富化

- DSL 扩展：眨眼 / 嘴型 / 多角度（左 / 右 / 背面）/ 装饰位
- 表情贴图（受角色 DNA 驱动）
- 桌宠"小动作"包：抖腿、看时间、伸懒腰
- 用户可"换装"（基于调色板的简单变体，不破坏角色）

#### 3.2.3 对话体验升级

- 流式输出的"打字机风格"可配置
- 长回复折叠 + "继续读"按钮
- 引用 / 强调 / 列表的轻量 Markdown 渲染
- 历史记录滚动浏览（按会话）

#### 3.2.4 严格模式

- 整体应用层级开关：开启后所有角色更克制（语言、话题、行为）
- 适合家用 / 共享电脑
- 默认关闭

#### 3.2.5 自动更新 + 崩溃上报（opt-in）

- electron-updater 接入 GitHub Releases
- 崩溃上报 opt-in，明确字段范围

### 3.3 不做

- 关系记忆 / 好感度
- 角色市场
- 多角色同时在桌

### 3.4 成功指标

- 7 日召唤率 ≥ 40%
- 平均会话长度 ≥ 5 轮
- 深度造人采用率 ≥ 20%（在新建角色中）
- 应用崩溃率 < 0.5%

### 3.5 退出门

- [ ] 自动更新通道稳定
- [ ] 深度造人 vs 快速造人的差异在用户问卷中显著（≥ 60% 觉得"明显更像"）
- [ ] 形象动作扩展不引入安全沙箱回归

## 4. v1.1 · 多角色 + 严格模式深化

### 4.1 主题

> "从'桌面养一只'升级到'桌面养几只，互不打扰'。"

### 4.2 关键能力

#### 4.2.1 多 Pet Window 并存

- 同时显示多只桌宠（上限例如 3 只）
- 每只独立的位置 / 状态机 / 闲置行为
- 唤起聊天时只激活被点击 / 快捷键聚焦的那只
- 互相之间在 MVP 不感知对方（v1.x 后期再考虑互动）

#### 4.2.2 角色仓库强化

- 标签、搜索、收藏
- 按使用频率 / 创建时间排序
- 缩略图列表 / 大图列表两种视图

#### 4.2.3 隐私快照

- 一键"隐身"：所有桌宠瞬间消失、对话窗最小化
- 用于演示 / 录屏 / 共享屏幕场景
- 全局快捷键

#### 4.2.4 严格模式深化

- 按"角色类别"独立设置严格度
- 儿童模式（v1.1 才放）

### 4.3 不做

- 角色之间多 Agent 对话
- 跨设备同步

### 4.4 成功指标

- 多角色用户的平均角色数 ≥ 2
- 严格模式开启用户比例 ≥ 15%

## 5. v1.2 · 关系养成

### 5.1 主题

> "让陪伴变得真有'我们'的痕迹。"

### 5.2 关键能力

#### 5.2.1 关系记忆

- 在 CharacterCard 旁增加 `relationship.<characterId, userId>` 文件
- 记录：好感度（轻量数值 + 阶段标签）、共同经历摘要、纪念时刻、用户的喜恶
- 用户可查看 / 编辑 / 清空（与 UserProfile 同一界面）

#### 5.2.2 关系事件

- 系统识别"值得记录的对话片段"（例如用户分享了重要决定 / 取得成就）
- 推送给用户："是否把这段记入和 X 的共同记忆？" 默认是
- 这些事件成为后续对话的隐藏上下文

#### 5.2.3 阶段化语气

- 关系不同阶段，角色对用户的称呼、用词、亲密度自适应（仍由角色 DNA 主导，不变成讨好型）
- 用户可冻结阶段或重置关系

#### 5.2.4 关系仪表盘

- 角色详情页新增"我们的关系"tab
- 关键数据可视化 + 关键时刻时间线

### 5.3 不做

- 多用户共享同一角色的关系
- 关系数据云同步

### 5.4 成功指标

- 关系功能开启用户的次月留存比未开启用户高 ≥ 15%
- 关系记录的"用户主动修改 / 删除"频率 < 20%（说明不打扰）

### 5.5 风险

- 风险：让用户对 AI 产生过强情感依赖
- 缓解：明确"AI 陪伴不替代人际"提示；提供"关系冷静"功能（一键重置但保留角色）

## 6. v1.3 · 主动陪伴

### 6.1 主题

> "从'我喊它出来'升级到'它在合适的时候出现一下'。"

### 6.2 关键能力

#### 6.2.1 工作感知（可选）

- 经用户授权，应用读取：当前活动窗口标题 / 剪贴板（脱敏后只用本地）/ 系统空闲时长
- 不上传，仅本地决定"现在桌宠是否应该出声"
- 默认关闭，必须每个数据源单独授权

#### 6.2.2 主动话语

- 桌宠基于自身性格 + 上下文偶尔说一句话（气泡形式，3 秒后自动消失）
- 触发条件配额：每小时最多 N 次，可配置
- 一键"安静一会"

#### 6.2.3 专注模式

- 用户进入"专注模式"，桌宠停止主动行为
- 番茄钟接入（轻量计时器）

### 6.3 不做

- 系统级通知（避免侵扰）
- 自动接管任务 / 操作其他应用

### 6.4 成功指标

- 主动陪伴开启用户的 7 日召唤率比关闭用户高 ≥ 20%
- "安静一会"使用频率 < 主动话语次数的 30%（说明频次合理）

### 6.5 风险

- 风险：太烦人 → 用户卸载
- 缓解：默认极保守（每小时 ≤ 1 次）；首次主动出声前先弹"试用一周"询问；任何主动行为都可单独关闭

## 7. v2.0 · 平台化与角色市场

### 7.1 主题

> "从'我的桌宠'升级到'我的桌宠 + 朋友们的桌宠'。"

### 7.2 关键能力

#### 7.2.1 角色导入导出标准

- `.bailin` 包格式正式标准化
- 含：CharacterCard、SpriteProgram、缩略图、可选示例对话
- 不含用户画像、关系数据

#### 7.2.2 社区分享渠道

- 官方推荐：GitHub 仓库 + 浏览页
- 一键导入：从分享链接 / 文件拖入
- 用户上传：要求登录、提交审核、内容安全检查

#### 7.2.3 信誉机制

- 简单的下载量 / 收藏量 / 举报
- 风险角色快速下架机制

#### 7.2.4 商业边界

- 整个市场免费
- 不接受付费角色（避免变成卖人格）
- 创作者可署名，不提供货币化

### 7.3 不做

- 在线对话 / 云端推理
- 跨设备同步用户数据（仍是本地）
- 多人聊天 / 用户社交

### 7.4 成功指标

- 用户中安装过 ≥ 1 个外部角色的比例 ≥ 30%
- 月活创作者 ≥ 100（社区指标）
- 严重内容事件 = 0

### 7.5 风险

- 风险：角色市场带来内容失控
- 缓解：审核（自动 + 人工）、举报、安全沙箱保证下载角色不能突破限制、对未授权 IP 严格下架

## 8. v2.x · 多模态

### 8.1 主题

> "从'读着 / 写着聊'升级到'像和真人一样自然的交互'。"

### 8.2 候选能力（按优先级）

#### 8.2.1 输入：语音

- 按住快捷键说话 → 转写 → 当作对话输入
- 转写在本地（Whisper 系列）或用户的云端服务（用户选）
- 角色仍以文字回复

#### 8.2.2 输出：语音

- 角色可选启用 TTS
- 默认关闭；启用前提示"声音不代表本人"
- TTS 服务用户自带（同 LLM Key 模型）

#### 8.2.3 表情贴图

- 角色在对话中根据情绪自动出表情（基于 DNA 限制库内的表情）
- 用户可关闭

#### 8.2.4 Live2D 风格扩展（探索）

- 在像素风之外，提供"风格切换"
- 仅作为可选风格，不替代默认像素风
- 这一步可能拆为独立次版本

### 8.3 风险

- 风险：声音模仿真人引起争议
- 缓解：永远不用真人声音克隆；TTS 用通用人声；明确标注；提供随时关闭

---

## 9. v3.0 · 跨端

### 9.1 主题

> "把桌宠带到更多桌面。"

### 9.2 关键能力

- macOS（菜单栏 + 桌面层，重点：透明窗口 + 全局快捷键）
- Linux（X11 / Wayland 兼容性单独评估）
- 移动端（探索性）：iOS / Android 上不做"桌宠"，但做"快速召唤"形态（一个常驻通知或灵动岛入口），与桌面端通过本地 LAN / 用户云盘同步角色

### 9.3 共享底层

- 核心包（CharacterRuntime / MemoryStore / SpriteCodeGen / SafetyPolicy）已在 MVP 抽离
- 渲染层（Web 技术栈）天然跨端
- 平台抽象层负责窗口 / 通知 / 加密 / 全局快捷键

### 9.4 不做

- Web 版（用户拿不到桌面在场感，价值损失大）
- 浏览器扩展形态（与桌宠定位冲突）

## 10. 不在路线图（明确放弃）

| 项目 | 原因 |
| --- | --- |
| 自己托管 LLM / 端侧大模型 | 与开源 + 自带 Key 路线冲突 |
| 真人语音克隆 | 法律 / 伦理 / 品牌风险过大 |
| 角色付费售卖 | 让"创造一个人格"变成商品，损害产品意义 |
| 角色之间的剧情系统 | 偏离"桌面陪伴 + 思维顾问"核心 |
| 全屏接管 / 桌宠"占领"桌面模式 | 与"配角而非主角"原则冲突 |
| 替用户操作其他应用 | Agent 自动化与桌宠产品边界混乱，安全风险 |

## 11. 不可逆决策清单（影响后续）

| 决策 | 阶段 | 后果 |
| --- | --- | --- |
| CharacterCard / SpriteProgram Schema v0.1 | MVP | 后续仅加字段不减；不兼容时必须出迁移器 |
| Web Worker 沙箱作为渲染唯一通道 | MVP | 一旦放开就回不去 |
| 用户 Key 永不出本机 | MVP | 任何"云推理"功能都不能直接接 |
| 不付费角色市场 | v2.0 | 之后不引入付费角色 |
| 不做语音克隆 | v2.x | 永远不做真人声音 |
| 桌宠默认是配角 | 全程 | 任何"全屏 / 抢焦点"功能都不被接受 |
